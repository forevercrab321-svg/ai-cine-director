# 多帧关键帧视频生成系统

## 概述
实现类似 Runway/Kling 的多帧链接生成，显著提升角色一致性和叙事能力。

## 当前 vs 目标

### 当前 (单图→视频)
```
[图A] → 生成视频 → [图A']
问题: 下一段视频可能角色变化
```

### 目标 (多帧链接)
```
[图A] ──→ 视频1 ──→ [图A']
  │
  └─[图B] ──→ 视频2 ──→ [图B']
      │
      └─[图C] ──→ 视频3 ──→ [图C']
```

## 实现方案

### 前端改动
1. 添加"多关键帧模式"开关
2. 每帧可选择:
   - 新建图像
   - 使用上一帧的尾帧
   - 上传自定义图像
3. 帧预览和时间线

### 后端改动
1. 新增 API: `/api/generate-chain`
2. 自动提取上一段视频的尾帧
3. 实现帧链接逻辑
4. 优化 credit 计算

### 一致性增强
- 每帧自动继承前帧角色锚点
- 强制相似度提示词
- 首帧→尾帧插值

## 技术栈
- Replicate: first_frame_image + last_frame_image
- Kling: support for start/end frames
- 自研: 尾帧提取模块

## 优先级
1. P0: 单图→视频→尾帧提取→下一段视频 (链式)
2. P1: 手动选择关键帧
3. P2: 完整时间线UI
