## 🐛 BUG #1: Story Entities Not Persisted to Prompt Chain
**Severity**: 🔴 CRITICAL
**Impact**: Character descriptions are lost between Gemini generation and video creation

### Issue
- Gemini returns `story_entities` in structured output
- These entities are converted to frontend format BUT missing `is_locked` field
- Video generation endpoints don't receive locked cast information
- Characters can morph, age-swap, or be replaced

### Root Cause
```typescript
// In api/index.ts line ~2385
const story_entities = entitiesRaw.map((e: any) => ({
    id: crypto.randomUUID(),
    type: normalizeEntityType(e.type),
    name: sanitizePromptInput(e.name || 'Unknown', 120) || 'Unknown',
    description: sanitizePromptInput(e.description || '', 1200),
    is_locked: normalizeEntityType(e.type) === 'character'  // ★ AUTO-LOCK all characters
}));
```

**Problem**: `is_locked` computed automatically, not from Gemini. Need to make Gemini include it.

### Fix
1. Add `is_locked` field to `geminiResponseSchema`
2. Instruct Gemini: "Set is_locked=true for protagonist characters"
3. Pass locked entities to video generation endpoints

## 🐛 BUG #2: Character Anchor Not Embedded in Video Prompts
**Severity**: 🔴 CRITICAL
**Impact**: Video frames don't know character identity - faces drift between scenes

### Issue
- Character anchor (e.g., "Asian woman, 25y, long black hair, white qipao") defined at story level
- Image generation receives it via `appendLockedCastToPrompt()`
- BUT video prompts generated by Gemini are SEPARATE and don't include anchor
- Video model (Hailuo, Kling, etc.) doesn't see character description

### Root Cause
```typescript
// In Gemini prompt template: video_prompt is JUST motion description
// "Camera tilts up slowly. Woman raises hand to window."
// MISSING: Character identity anchor!
```

### Fix
1. Inject character anchor into ALL video_prompts
2. Append locked cast directive to video_prompt in `/api/replicate/predict`
3. Pass `characterAnchor` from Gemini response to video endpoints

## 🐛 BUG #3: Non-Human Species Not Forced into Story Entities
**Severity**: 🔴 CRITICAL
**Impact**: Animals become humans in final videos

### Issue
- System detects non-human species (cat, dog, rabbit, bear) from story idea
- Creates directive: `[NON-HUMAN CHARACTER LOCK: The protagonist/cast are cat characters...]`
- BUT Gemini doesn't read this from `story_entities.description` when generating videos
- Replicate receives generic motion prompt without species lock

### Root Cause
```typescript
// detect NonHumanCharacterGuide() runs AFTER Gemini response
// So Gemini doesn't know to output animal-specific descriptions
```

### Fix
1. Run species detection BEFORE Gemini call
2. Add to system instruction: "If animals detected, set description='A [SPECIES] character with [ANATOMY]'"
3. Ensure `story_entities[0].description` includes species-specific anatomy keywords

## 🐛 BUG #4: is_locked Field Missing from Gemini Schema
**Severity**: 🔴 CRITICAL
**Impact**: Locked cast directive not enforced in video generation

### Issue
- Backend code assumes Gemini returns `is_locked: true/false`
- But Gemini schema doesn't include this field
- All entities marked `is_locked: true` by DEFAULT (never false)
- No way to distinguish "locked protagonist" from "optional background character"

### Root Cause
```typescript
const geminiResponseSchema = {
    properties: {
        story_entities: {
            items: {
                properties: {
                    type: { type: Type.STRING },
                    name: { type: Type.STRING },
                    description: { type: Type.STRING },
                    // ★ MISSING: is_locked field!
                }
            }
        }
    }
}
```

### Fix
- Add `is_locked: { type: Type.BOOLEAN }` to schema
- Gemini will then return `is_locked: true` for protagonist only

## 🐛 BUG #5: Negative Prompt Missing Character Protection
**Severity**: 🟠 HIGH
**Impact**: Flux/Replicate can ignore character consistency constraints

### Issue
- Image/video models have negative prompts (e.g., "blurry, bad quality")
- But no negative prompt protects character identity
- Should include: "Don't change face, don't replace character, don't age-swap"
- Currently only in positive prompt (weaker enforcement)

### Root Cause
```typescript
// Gemini returns image_prompt but NO negative_prompt
// Frontend falls back to generic negatives without identity protection
```

### Fix
1. Add `negative_prompt` to Gemini schema
2. Instruct Gemini: "Return negative_prompt: 'altered identity, different person, age change, morph, identity drift'"
3. Pass negative_prompt to Replicate in `/api/replicate/predict`

## 🐛 BUG #6: Director-Level Prompting Insufficient
**Severity**: 🟠 HIGH
**Impact**: Generated storyboards lack cinematic excellence

### Issue
- Gemini system instruction focuses on story structure only
- Missing cinematography mastery directives:
  - Composition rules (rule of thirds, depth layering)
  - Lighting language (chiaroscuro, color temperature)
  - Camera language (angle psychology, movement storytelling)
  - Visual scale and atmosphere
- Result: Technically correct but visually uninspired storyboards

### Root Cause
```typescript
const systemInstruction = `You are a Hollywood Screenwriter...`
// Focuses on: three-act structure, scene uniqueness, character development
// MISSING: Spielberg's framing, Nolan's visual architecture, Villeneuve's scale
```

### Fix
1. Add "Master Cinematographer" persona to system instruction
2. Include visual excellence pillars:
   - COMPOSITION: rule of thirds, leading lines, depth, negative space
   - LIGHTING: chiaroscuro, color storytelling, practical sources, atmosphere
   - CAMERA LANGUAGE: angle psychology, movement intention
   - SCALE & DEPTH: cinematic breathing room
3. Examples: "Establish shot: bird's-eye of vast landscape. Cut to extreme close-up of character's eye."

## 🐛 BUG #7: Image Prompt Duplication Across Shots in Scene
**Severity**: 🟠 HIGH
**Impact**: Wasted tokens, potential identity confusion

### Issue
- First shot in scene: detailed image_prompt (good for establishing shot)
- Subsequent shots: image_prompt should be "" (empty) for video chain continuity
- Currently: Gemini may return image_prompt for ALL shots
- When passed to video model: all shots re-described = breaks frame continuity

### Root Cause
```typescript
// Gemini doesn't know to empty image_prompt for shots 2+
// System instruction doesn't specify this rule
```

### Fix
1. Add to system instruction:
   ```
   "image_prompt strategy:
   - Shot 1 in scene: Full detailed prompt
   - Shot 2+ in same scene: image_prompt = "" (empty string, use video_prompt only)"
   ```
2. Validate in backend: `if (shot.shot_index > 1) image_prompt = ""`
